Проверка номеров для покупки: Yandex ID, Pay карта, верификация, блокировки

📋 Логика проверки

✨
ЧИСТЫЙ (clean) Нет ID, нет Pay карты. Рекомендуем к покупке — идеальный номер
✅
ХОРОШИЙ (pay_verif) Pay карта открыта + подтверждено в Госуслугах. Рекомендуем к покупке
⚠️
НУЖНА ВЕРИФИКАЦИЯ (pay_no_verif) Pay карта есть, но Госуслуги не подтверждены. Купить можно, требуется верификация по дате рождения бывшего владельца
🛑
СТОП (blocked) Номер заблокирован. НЕ ПОКУПАТЬ
❓
Только ID (id_no_pay) Есть аккаунт Яндекса, но нет Pay карты. Нейтральный статус

🔧 Параметры проверки:

  • id — есть ли Yandex ID (аккаунт в Яндексе)
  • pay — есть ли открытая карта Yandex Pay
  • гу — подтверждено ли в Госуслугах (по наличию даты рождения)
  • дд.мм.гггг — дата рождения из Госуслуг (нужна для верификации)
  • блок — есть ли ограничения/блокировка на номере
💡 Ключевое правило верификации:
Верифицирован = Pay карта открыта + найдена дата рождения в Госуслугах.
Если Pay есть, но даты нет — значит при верификации была указана неверная дата или Госуслуги не подтверждены.
Дата рождения нужна для повторной верификации номера через Госуслуги.
📅 Правило поиска даты рождения:
Дата рождения ищется для ВСЕХ номеров, кроме чистых (где нет ID и нет Pay).
То есть ищем дату если: есть Pay карта ИЛИ есть ID ИЛИ есть блокировка.
Дата нужна для всех не-чистых номеров — независимо от статуса верификации.
